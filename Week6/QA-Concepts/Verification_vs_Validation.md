# Verification vs Validation

## Verification

Verification checks whether the software is being developed according to the specified requirements.

**Question answered:** Are we building the product right?

### Examples
- Requirement Review
- Design Review
- Code Review
- Walkthroughs
- Inspections

---

## Validation

Validation checks whether the developed software meets the customer's needs.

**Question answered:** Are we building the right product?

### Examples
- Unit Testing
- Integration Testing
- System Testing
- User Acceptance Testing (UAT)

---

## Difference

| Verification | Validation |
|--------------|------------|
| Static process | Dynamic process |
| No code execution | Code execution required |
| Finds defects early | Ensures software works correctly |
| Done before testing | Done during testing |

## Interview Tip

**Verification** = Prevent defects.

**Validation** = Detect defects.
